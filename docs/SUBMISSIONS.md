# Инструкции по сдаче лабораторных работ

## Как сдавать лабораторные?

### Способ сдачи: Fork + Pull Request

Каждая лабораторная работа сдаётся через **Pull Request (ПР)** из вашего форка репозитория.

### Пошаговая инструкция

1. Форкай репозиторий
2. Клонируй свой форк локально
```bash
git clone https://github.com/<ТВ-ЛОГИН>/cuda-cource-2025.git
cd cuda-cource-2025
```

3. Создай свою папку для работ
В папке `works/` создай папку с твоим именем/логином:
```bash
mkdir works/<ТВ-ЛОГИН>
```
Подробнее про шаблон лабораторных и структуру папок см. в [**FOLDER_STRUCTURE.md**](FOLDER_STRUCTURE.md) и [**STUDENT_TEMPLATE.md**](STUDENT_TEMPLATE.md).

4. Работай с лабораторными

5. Коммитьте ваши изменения
```bash
git add works/<ТВ-ЛОГИН>/
git commit -m "Add Lab X: <название лабы>"
git push origin master
```

6. Создай Pull Request
- Перейди в свой форк на GitHub
- GitHub автоматически предложит создать PR
- Или создай PR вручную: в исходном репозитории нажми **Pull requests** → **New pull request** → **compare across forks**
- Выбери: `base: comiam/cuda-cource-2025:master` ← `head: <ТВ-ЛОГИН>/cuda-cource-2025:master`
- Добавь описание:
  ```
  ## Лабораторная N: <название>
  
  - [ ] Задача выполнена полностью
  - [ ] Код скомпилирован без ошибок
  - [ ] Код протестирован
  
  ### Что было сделано:
  - ...
  - ...
  ```

#### 7. Дождись проверки
- Преподаватель проверит ваш PR
- Если будут замечания, внеси исправления в свою ветку и запуши их
- Изменения автоматически добавятся в PR

### Правила оформления ПР

✅ **Делай:**
- Понятное название: `Фамилия А.В. <группа>. Lab N: <название>`
- Подробное описание с примерами запуска
- Чистый, отформатированный код
- Коммиты с понятными сообщениями

❌ **Не делай:**
- Не добавляй бинарные файлы и объектные файлы (`.o`, `.out`, и т.д.)
- Не коммитьте большие файлы с моделями/данными
- Не используй бранчи с неинформативными названиями (`test`, `work`, `fix1`)

---

**Важно:** Каждый PR должен содержать работу **только одной лабораторной**. Если у тебя готово несколько лаб, создавай несколько ПР.
